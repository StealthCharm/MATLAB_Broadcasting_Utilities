# +broadcast
This small package was written so that function authors can implement broadcasting semantics without the overhead associated with cellfun(). cellfun() negatively impacts performance more than for-loops in most instances and since cellfun() does not support broadcasting, it may additionally require that inputs be expanded. Duplication of elements to allow cellfun() calls to execute also act to impact performance. The functions act to produce index maps relating the operands to one another so that fast, simplified for-loops can be used in its place by adopting a few simple working patterns as "primitives".

For more indepth discussion of these patterns, along with the explanation and reasoning behind most of the design decisions made in this package, see the `Examples` directory, as it exhaustively covers both.

To reduce the inconvenience of writing the boiler plate these primitive working patterns require, snippets have been created using the extension points in the new MATLAB desktop environment, the `extensions.json` defining them can be found in the `resources` directory. *NOTE: for tab completion of the snippets to work, the file must be on the MATLAB path; if the current heirarchy of the package is left unmodified, you will not be able to use the snippets; if you would like to use them you can move the `extensions.json` file out of the resources and onto you MATLAB path. This is known and intentional, their presences is meant to be optional, or inspire you to customize them to your likiing.*
